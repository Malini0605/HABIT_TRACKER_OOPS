
# Habit Tracker – Console Based OOP Project

A console-based **Habit Tracker application built using Python and Object-Oriented Programming (OOP)**.
This project was designed and implemented step-by-step to practice **software design, OOP principles, and system modeling**.

The goal of this project was not only to write code, but also to **understand the full development process**, starting from system design to implementation.

---

## Project Motivation

While building this project, I spent significant time understanding how a real system should be designed before writing code.
I started by identifying the **entities and relationships**, drawing an **ER diagram**, and then converting that design into **OOP classes and program logic**.

This helped me understand how:

* Database design connects to application logic
* Real-world systems can be modeled using classes and objects
* Good software starts with **proper planning and structure**

It was a valuable learning experience and gave me deeper insight into **object-oriented system design**.

---

## Features

* User registration and login (email + password authentication)
* Multi-user support
* Habit creation and tracking
* Automatic streak tracking
* Reminder system
* Journal entries for reflections
* Reward system based on streak milestones
* Community posts for sharing progress
* Menu-driven console interface

---

## Entities Used (From ER Design)

This project was first designed using an **Entity Relationship model**.

Entities identified:

* User
* Habit
* Reminder
* HabitProgress
* Journal
* Reward
* CommunityPost

The ER diagram helped in converting the design into the following **OOP classes**:

* `User`
* `Habit`
* `Reminder`
* `HabitProgress`
* `Journal`
* `Reward`
* `CommunityPost`
* `HabitTrackerApp`
